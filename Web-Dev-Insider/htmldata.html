HTML [Hyper Text Markup Language]

HyperText Markup Language (HTML) is the standard markup language for documents designed to be displayed in a web browser. It defines the content and structure of web content. It is often assisted by technologies such as Cascading Style Sheets (CSS) and scripting languages such as JavaScript. Web browsers receive HTML documents from a web server or from local storage and render the documents into multimedia web pages. HTML describes the structure of a web page semantically and originally included cues for its appearance. HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects such as interactive forms may be embedded into the rendered page. HTML provides a means to create structured documents by denoting structural semantics for text such as headings, paragraphs, lists, links, quotes, and other items. HTML elements are delineated by tags, written using angle brackets. Tags such as directly introduce content into the page. Other tags such as surround and provide information about document text and may include sub-element tags. Browsers do not display the HTML tags but use them to interpret the content of the page. HTML can embed programs written in a scripting language such as JavaScript, which affects the behavior and content of web pages. The inclusion of CSS defines the look and layout of content. The World Wide Web Consortium (W3C), former maintainer of the HTML and current maintainer of the CSS standards, has encouraged the use of CSS over explicit presentational HTML since 1997.[2] A form of HTML, known as HTML5, is used to display video and audio, primarily using the element, together with JavaScript.

What is an HTML Element

An HTML element is a type of HTML (HyperText Markup Language) document component, one of several types of HTML nodes (there are also text nodes, comment nodes and others).[vague] The first used version of HTML was written by Tim Berners-Lee in 1993 and there have since been many versions of HTML. The current de facto standard is governed by the industry group WHATWG and is known as the HTML Living Standard. An HTML document is composed of a tree of simple HTML nodes, such as text nodes, and HTML elements, which add semantics and formatting to parts of document (e.g., make text bold, organize it into paragraphs, lists and tables, or embed hyperlinks and images). Each element can have HTML attributes specified. Elements can also have content, including other elements and text.

Web Browsers

A web browser is an application for accessing websites. When a user requests a web page from a particular website, the browser retrieves its files from a web server and then displays the page on the user's screen. Browsers are used on a range of devices, including desktops, laptops, tablets, and smartphones. In 2020, an estimated 4.9 billion people have used a browser.[1] The most-used browser is Google Chrome, with a 64% global market share on all devices, followed by Safari with 19%.[2] A web browser is not the same thing as a search engine, though the two are often confused.[3][4] A search engine is a website that provides links to other websites. However, to connect to a website's server and display its web pages, a user must have a web browser installed.[5] In some technical contexts, browsers are referred to as user agents.

Meta Tag

The meta tag defines metadata about an HTML document. Metadata is data (information) about data. meta tags always go inside the head element, and are typically used to specify character set, page description, keywords, author of the document, and viewport settings. Metadata will not be displayed on the page, but is machine parsable. Metadata is used by browsers (how to display content or reload page), search engines (keywords), and other web services. There is a method to let web designers take control over the viewport (the user's visible area of a web page), through the meta tag (See "Setting The Viewport" example below).

Setting the Viewport

The viewport is the user's visible area of a web page. It varies with the device - it will be smaller on a mobile phone than on a computer screen. You should include the following meta element in all your web pages: meta name="viewport" content="width=device-width, initial-scale=1.0" This gives the browser instructions on how to control the page's dimensions and scaling. The width=device-width part sets the width of the page to follow the screen-width of the device (which will vary depending on the device). The initial-scale=1.0 part sets the initial zoom level when the page is first loaded by the browser.

HTML Article Element

The article element in HTML represents a self-contained, independent piece of content that could be distributed or reused elsewhere. It is typically used for content like blog posts, news articles, forum posts, or other sections that make sense on their own.

Key Features:

  • Self-contained: An article should make sense by itself without requiring additional content to be understood.
  • Reusable: The content inside an article can be shared, syndicated, or reused in other contexts.
  • Semantics: It helps improve the accessibility and search engine optimization (SEO) of a website by defining what content is considered a standalone section.

    • HTML Header Element

    The header tag in HTML is used to define the introductory content or a group of navigational links for a section or an entire webpage. It is typically placed at the top of a page, section, or article and can contain elements like headings (h1 to h6), logos, navigation menus, or any introductory information relevant to the content that follows.

    Key Features:

  • Can be used multiple times: You can use the header tag in different sections of your page, such as inside an article or section.
  • Not just for the top of the page: Although it's often found at the top, a header can appear within a specific section to introduce that content.
  • Semantic element: It helps with the organization and accessibility of the content, providing context to assistive technologies and search engines.

    • HTML Footer Element

    The footer tag in HTML defines the footer section of a webpage or a specific section within it. It typically contains information about the author, copyright data, links to related content, or other metadata related to the page or section.

    Key Features:

  • Appears at the bottom: It usually appears at the bottom of the page or section, though it can also be used in other places if necessary.
  • Contains supplementary information: It often holds elements like contact information, disclaimers, copyright notices, links to privacy policies, or navigational links.
  • Can be used multiple times: Like the header, the footer can be used multiple times within a webpage, for example in different sections or articles.

    • HTML Nav Element

    The nav element in HTML is used to define a section of the page intended for navigation. It contains links to other parts of the website or different pages, typically used for main site navigation menus, sidebars, or links within a page (like a table of contents).

    Key Features:

  • Navigation links: The nav element holds links to important sections, pages, or content, such as menus or internal page links.
  • Can be used multiple times: It can appear multiple times on a page, like for both main navigation and secondary navigation (e.g., in the footer).
  • Semantics: It helps browsers and assistive technologies (like screen readers) understand that the links within are for navigation purposes.

    • HTML Aside Element

    The aside element in HTML is used to represent content that is tangentially related to the main content of the webpage. It is typically used for sidebars, pull quotes, advertisements, or other supplementary information. Content inside an a is not essential for understanding the main content, but it provides additional context or related information.

    Key Features:

  • Supplementary content: It often holds related information like links, advertisements, or author bios, which are not part of the main flow but enhance the user experience.
  • Not required for the main content: Information inside the aside should be skippable without losing the overall meaning of the main content.
  • Can be used inside articles or sections: The aside element can appear either within an article (to present relevant information) or outside it (as a general sidebar).

    • HTML Figure Element

    The figure element in HTML is used to encapsulate media, such as images, diagrams, illustrations, code snippets, or other content, along with an optional caption. It groups media together with a caption (figcaption) to provide additional context or descriptions. The figure element is typically used for content that is referenced in the main content but can stand alone if needed.

    Key Features:

  • Media and caption pairing: The figure element groups media with a caption, which is defined by the figcaption element.
  • Self-contained: The content inside the figure can be moved or placed anywhere in the document without affecting its meaning.
  • Optional caption: The figcaption tag can be used within the figure to provide a description of the media.

    • HTML Figcaption Element

    The figcaption element in HTML is used to provide a caption or description for the content contained in a figure element. It helps explain or give context to the media (like images, diagrams, or code snippets) within the figure.

    Key Features:

  • Used with figure: The figcaption is always used inside the figure element to describe its content.
  • Positioning: It can be placed either before or after the media within the figure /figure
  • Enhances accessibility: It improves the accessibility and understanding of the media by giving users a text-based description of what they are seeing.

    • Navigation Bar

    The nav tag in HTML defines a section of the webpage specifically for navigation links. These links usually lead to other parts of the website or external resources. The nav element is commonly used for menus, tables of contents, or any set of navigational links that users can use to navigate through a website.

    Key Features:

  • Navigation purpose: The primary function is to organize navigational links.
  • Can contain lists or links: It usually contains an unordered list (ul) of links (a elements), but it can contain any navigation-related elements.
  • Can be used multiple times: It can appear more than once on a webpage, such as for main navigation at the top and supplementary navigation in the footer.
  • Semantic: It helps improve accessibility by signifying to screen readers and search engines that the links are used for navigation.

    • Divisions

    The HTML div tag defines sections in HTML documents, serving as containers styled with CSS or controlled with JavaScript. It’s easily customized using class or id attributes and can contain various content.

    Div tag Usage:

  • The div tag is the block-level tag.
  • It is used for applying styles and layout structure div and allows us to manipulate and position content through CSS.
  • It also divides content into logical sections, aiding in the readability and maintainability of the code.
  • As we know, a Div tag is a block-level tag containing the entire width. Hence, every div tag will start from a new line and not the same line.

    • Tables in HTML

    The table element in HTML is used to display data in a tabular format, consisting of rows and columns. It is commonly used to organize and present information like schedules, pricing charts, comparisons, or any structured data. A table is made up of several elements like tr (table row), td (table data cell), and th (table header cell) to structure the content properly.

    Key Elements of a Table:

  • table: Defines the overall table structure.
  • tr: Defines a row within the table.
  • th: Defines a header cell in a table, which is usually bold and centered.
  • td: Defines a standard data cell in a table.
  • caption (optional): Provides a title or description for the table.
  • thead, tbody, tfoot (optional): Group rows together for headers, body, and footers respectively.

    • Alignment

    In HTML, alignment refers to the positioning of text, images, or other elements relative to their container. Alignment can be applied to content horizontally (left, center, right) or vertically (top, middle, bottom). HTML and CSS offer various ways to control alignment for elements

    1. Text Alignment (Horizontal)

    You can align text content using the text-align CSS property.

  • Left: Aligns text to the left (default for most browsers).
  • Center: Centers the text.
  • Right: Aligns text to the right.
  • Justify: Stretches the text so that it aligns with both the left and right edges.

    • 2. Image Alignment (Horizontal)

    For aligning images, you can use the float or text-align properties.

  • Float: The float property can align images to the left or right of the content

    • 3. Vertical Alignment

    Vertical alignment refers to aligning elements vertically within their container. The vertical-align property is often used for inline or table-cell elements.

  • Top: Aligns the content to the top of the container.
  • Middle: Aligns the content in the middle.
  • Bottom: Aligns the content to the bottom of the container.

    • 4. Aligning Block Elements

    Block elements like div, p, or section are usually aligned with margin, especially for centering.

    Summary of Properties:

  • text-align: For horizontal alignment of text and inline elements (left, center, right, justify).
  • float: For floating images or elements to the left or right.
  • vertical-align: For vertical alignment in inline or table-cell elements (top, middle, bottom).
  • margin-auto: For centering block-level elements horizontally.

    • Borders in HTML

    In HTML, borders are used to outline or frame elements, enhancing the visual structure and design of a webpage. Borders are primarily controlled using CSS (Cascading Style Sheets), which allows you to set border styles, widths, and colors for HTML elements.

    Key Properties for Borders in CSS:

    1. border: A shorthand property for setting the border width, style, and color in one line.
    2. border-width: Specifies the thickness of the border.
    3. border-style: Defines the style of the border (e.g., solid, dashed, dotted).
    4. border-color: Sets the color of the border.
    5. border-radius: Applies rounded corners to the border.

    Forms in HTML

    Forms in HTML are used to collect user input, such as text, choices, or file uploads, and submit this data to a server for processing. Forms are fundamental for interactive elements on websites, like sign-up forms, search fields, and feedback forms.

    Key Elements of an HTML Form:

    1. form: The container element for form elements. It wraps all form controls and specifies how and where the form data should be submitted.
    2. input: A versatile element used for various types of user input, such as text, password, checkbox, radio, and file uploads.
    3. label: Defines a label for an input element. It helps improve accessibility by associating text labels with form controls.
    4. textarea: Provides a multi-line text input field, useful for longer text entries like comments or messages.
    5. select: Creates a dropdown list of options for users to choose from.
    6. button: Defines a clickable button, which can be used to submit the form or trigger other actions.
    7. fieldset: Groups related elements within a form, typically used with legend to provide a caption for the grouped elements.
    8. legend: Provides a caption or title for a fieldset.
    Key Attributes:
  • action: Specifies the URL where the form data will be sent when the form is submitted.
  • method: Defines the HTTP method (usually GET or POST) for sending form data.
  • name: Names each form control so that the server can identify and process the data.
  • id: Uniquely identifies form elements, often used with label for accessibility.
  • required: Indicates that a form field must be filled out before submitting the form.
  • rows and cols: Define the size of a textarea.
    • Form Controls:
  • input type="text": For single-line text input.
  • input type="password": For password fields.
  • input type="radio": For radio buttons.
  • input type="file": For file uploads.
  • button: Can be used as a submit button or for other actions.

    • Required

    In HTML forms, the required attribute is used to specify that a particular input field must be filled out before submitting the form. When the required attribute is added to an input element, the form will not be submitted until the user provides a value for that field.

    How to Use the required Attribute: Add required to Input Elements: You simply include the required attribute in the form control elements where input is mandatory.

    Key Points:

    1. Validation: The required attribute triggers browser-based validation. If the field is left empty, the browser will prevent form submission and display an error message.
    2. Error Message: The default error message displayed by the browser varies by browser and locale, but it usually indicates that the field is required.
    3. Multiple Fields: You can apply the required attribute to multiple fields in a form to ensure that all necessary information is provided.
    4. Types of Inputs: The required attribute can be used with various types of input elements, including input, select, and textarea.

    Placeholder

    The placeholder attribute in HTML is used to provide a hint or example text inside an input field or textarea that disappears when the user starts typing. It helps users understand what type of information is expected in a field and provides a visual cue about the input format.

    Key Points:

    1. Usage: The placeholder attribute is primarily used with input and textarea elements.
    2. Purpose: It provides a brief description or example of the required input, helping to guide users on what to enter.
    3. Appearance: The placeholder text appears inside the input field in a lighter color than the input text. It disappears when the user starts typing.
    4. Accessibility: While placeholders can be helpful, they should not replace labels (label /label)
    Key Attributes:
  • placeholder: Sets the placeholder text for the input or textarea element.
  • required: Ensures that the field must be filled out before submission.
    • Limitations:
  • Not a Substitute for Labels: Placeholder text should not replace labels. Labels are important for accessibility and provide more persistent context for form fields.
  • Text Disappears: Placeholder text disappears when the user starts typing, so it should not be the sole source of information for users about what to input.

    • Pattern

    The pattern attribute in HTML is used to specify a regular expression (regex) that an input field's value must match in order for the form to be submitted. This attribute is used with input elements to enforce specific input formats, such as phone numbers, email addresses, or passwords

    Key Points:

    1. Regular Expressions: The value of the pattern attribute is a regular expression that defines the allowed input format.
    2. Validation: If the user input does not match the regular expression, the form will not be submitted, and the browser will display an error message.
    3. Client-Side Validation: The pattern attribute provides client-side validation, which can enhance user experience by catching input errors before the form is submitted.
    4. Compatibility: The pattern attribute is supported by modern browsers, but it is always a good practice to also validate input on the server side to ensure data integrity.
    Key Attributes:
  • pattern: Defines the regular expression that the input field’s value must match.
  • title: (Optional) Provides a message that will be displayed when the input does not match the pattern. This helps users understand the expected format.

    • Autofocus

    The autofocus attribute in HTML is used to automatically focus an input field or textarea when a webpage loads. This means that the specified field will be active and ready for user input as soon as the page is displayed, which can enhance user experience by reducing the number of clicks needed to start typing.

    Key Points:

    1. Automatic Focus: The autofocus attribute is added to a single form element to make it the active element when the page loads.
    2. Single Usage: Only one element in a document can have the autofocus attribute. If multiple elements have autofocus, only the first one will be focused.
    3. Usage: It is often used in forms to streamline the user experience, especially on forms where immediate user input is expected.
    Notes:
  • Form Submission: When using autofocus on forms, be mindful of how it affects the user experience, especially on mobile devices or when multiple forms are present.
  • Accessibility: Ensure that automatic focus does not interfere with users’ ability to navigate the page, particularly for those using screen readers or other assistive technologies.

    • Audio

    The audio element in HTML is used to embed audio content on a webpage. It provides a way to play audio files directly in the browser without requiring third-party plugins. You can control audio playback using various attributes and controls.

    Key Attributes and Features:

    1. src: Specifies the URL of the audio file to be played.
    2. controls: Displays playback controls like play, pause, and volume.
    3. autoplay: Automatically starts playing the audio when the page loads.
    4. loop: Repeats the audio once it finishes playing.
    5. muted: Mutes the audio.
    6. preload: Specifies how the audio file should be loaded when the page loads. It can be auto, metadata, or none.

    Video

    The video element in HTML is used to embed video content directly into a webpage. It supports a variety of video formats and provides attributes and controls for managing video playback.

    Key Attributes and Features:

    1. src: Specifies the URL of the video file to be played.
    2. controls: Displays playback controls such as play, pause, volume, and fullscreen.
    3. autoplay: Automatically starts playing the video when the page loads.
    4. loop: Repeats the video once it finishes playing.
    5. muted: Mutes the video’s audio.
    6. poster: Provides an image to be shown before the video starts playing.
    7. preload: Specifies how the video file should be loaded when the page loads. Options are auto, metadata, and none.

    Track Element

    The track element in HTML is used to specify text tracks for video and audio elements. Text tracks can provide subtitles, captions, descriptions, or other metadata that enhance accessibility and usability for multimedia content. The track element is used inside the video or audio element to add these textual overlays.

    Key Attributes of the Element:

    1. src: The URL of the text track file. This file usually has a format like WebVTT (.vtt).
    2. kind: Specifies the type of text track. Common values include:
      • subtitles: Provides translated subtitles.
      • captions: Provides captions for the deaf and hard-of-hearing, including non-speech information.
      • descriptions: Provides audio descriptions for visually impaired users.
      • chapters: Provides chapter titles or sections.
    3. srclang: Specifies the language of the track using a language code (e.g., en for English, es for Spanish).
    4. label: Provides a user-readable name for the track. This label is used in track selection menus.
